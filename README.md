# 井字棋博弈树构建 - 教学活动

这是一个交互式的教学工具，用于帮助学生理解和构建井字棋（Tic-Tac-Toe）的博弈树。

## 项目简介

本项目是一个基于 HTML、CSS 和 JavaScript 的交互式教学应用，旨在通过可视化方式帮助学生：
- 理解博弈树的概念和结构
- 学习如何从初始棋盘状态构建完整的博弈树
- 分析不同走法对游戏结果的影响
- 思考最优策略和 AI 设计

## 功能特性

### 🎯 核心功能

1. **棋盘库**
   - 提供多个不同状态的棋盘（A-K）
   - 棋盘随机排列，增加挑战性
   - 自动标记新下的棋子，便于识别变化

2. **博弈树构建**
   - 拖拽式操作：从左侧棋盘库拖拽棋盘到工作区
   - 节点连接：点击节点下方的连接点绘制箭头，建立父子关系
   - 节点移动：拖动节点调整位置，优化树的可视化布局
   - 节点删除：删除不需要的节点（初始节点 A 不可删除）

3. **验证功能**
   - 一键验证博弈树结构是否正确
   - 检查所有必需节点是否存在
   - 验证节点间的连接关系是否符合逻辑
   - 提供详细的错误和警告信息

4. **思考问题**
   - 胜负可能性分析
   - 最优策略推演
   - AI 设计思考

## 使用方法

### 快速开始

1. 直接在浏览器中打开 `活动2 博弈树构建.html` 文件
2. 页面加载后，初始棋盘 A 会自动添加到工作区
3. 从左侧棋盘库拖拽其他棋盘到工作区
4. 点击节点下方的连接点，拖动到目标节点创建连接
5. 构建完成后，点击"验证博弈树"按钮检查结构

### 操作说明

- **添加节点**：从左侧棋盘库拖拽棋盘到右侧工作区
- **移动节点**：在工作区中直接拖动节点到新位置
- **创建连接**：点击源节点下方的连接点，拖动到目标节点的上半部分
- **删除节点**：点击节点右上角的 × 按钮（初始节点 A 不可删除）
- **验证树结构**：点击"验证博弈树"按钮

## 博弈树结构

### 节点关系

- **根节点**：A（初始棋盘状态）
- **子节点关系**：
  - A → B, C, D
  - C → E, F
  - D → G, H
  - E → I
  - F → J
  - G → K

### 棋盘状态

每个棋盘用字母标识（A-K），代表井字棋游戏的不同状态。新下的棋子会用特殊颜色标记：
- O 棋子：青色标记
- X 棋子：红色标记

## 技术实现

- **前端技术**：纯 HTML、CSS、JavaScript
- **拖拽功能**：HTML5 Drag and Drop API
- **图形绘制**：SVG 绘制连接线和箭头
- **交互设计**：响应式布局，支持鼠标操作

## 教学建议

1. **引入阶段**：先介绍博弈树的基本概念
2. **实践阶段**：让学生自主构建博弈树，体验不同走法
3. **分析阶段**：引导学生分析胜负可能性和最优策略
4. **拓展阶段**：鼓励学生思考 AI 设计思路

## 注意事项

- 建议使用现代浏览器（Chrome、Firefox、Edge 等）
- 工作区中的节点可以自由移动，但连接关系需要正确建立
- 验证功能会检查所有必需的节点和连接关系
- 初始节点 A 是根节点，不能删除
